# 快速上手指南

本文档提供使用本 skill 的最简路径，帮助你快速开始产品发现流程。

---

## 场景 1：从零开始发现产品机会

**适用情况**：你有一个大致方向，但不确定具体做什么

**触发指令**：
```
我想要创建一个 [产品类型]，帮助 [目标用户] 解决 [某类问题]。
请帮我进行完整的产品发现流程，从机会发现开始，
一直到生成 PRD 和核心功能的 IntentSpec。
```

**示例**：
```
我想要创建一个教育科技产品，帮助大学生更好地准备求职面试。
请帮我进行完整的产品发现流程，包括：
1. 分析当前大学生面试准备的痛点
2. 评估市场机会大小
3. 定义目标用户画像
4. 设计解决方案
5. 编写 PRD 文档
6. 为核心功能编写 IntentSpec
```

**你将获得**：
- 高价值问题列表（基于情感分析聚类）
- RICE 评分表和市场规模估算
- 3 个典型 Persona 和 JTBD 分析
- Kano 分类和 MoSCoW 优先级
- 完整的 PRD 文档结构
- 核心功能的详细 IntentSpec

## 场景 2：已有产品想法，需要 PRD

**适用情况**：你已经确定了产品方向，需要结构化文档

**触发指令**：
```
我要做一款 [产品名称]，核心功能是 [功能描述]。
请帮我写一份完整的 PRD 文档，包括用户故事地图、
功能优先级、非功能需求等。
然后为 [某个核心功能] 写 IntentSpec。
```

**示例**：
```
我要做一款冥想助眠 APP，核心功能是帮助失眠用户快速入睡。
请帮我写一份完整的 PRD 文档，包括：
- 产品愿景和目标用户
- 用户旅程地图（从发现 APP 到养成习惯）
- 功能优先级（P0/P1/P2）
- 非功能需求

然后为"智能睡眠引导"这个功能写详细的 IntentSpec，
要包含所有边缘情况（如用户中途睡着、网络中断等）。
```

**你将获得**：
- 完整 PRD 文档（可直接用于开发）
- 核心功能的 IntentSpec（可直接交给开发或 AI 编码）

## 场景 3：优化现有功能

**适用情况**：已有产品，某个功能需要改进

**触发指令**：
```
我们的 [产品] 的 [功能] 目前 [问题描述]，
[关键指标] 只有 [当前值]。
请帮我写一份 IntentSpec，目标是提升到 [目标值]，
要包含所有可能的边缘情况和验证方案。
```

**示例**：
```
我们的 SaaS 产品的"邀请团队成员"功能目前转化率很低，
只有 15% 的用户成功邀请同事。
请帮我写一份 IntentSpec，目标是提升到 40%，
要包含：
- 当前问题分析（为什么转化率低）
- 用户目标（用户邀请同事时真正想要什么）
- 详细的边缘情况（如邮箱无效、对方已注册、邀请链接过期等）
- 验证方案（A/B 测试、上线监控）
```

**你将获得**：
- 基于数据的 IntentSpec
- 全面的边缘情况覆盖
- 可执行的验证方案

## 场景 4：学习 PRD/IntentSpec 写作

**适用情况**：你想学习如何写好产品文档

**触发指令**：
```
请给我展示一个完整的 [PRD/IntentSpec] 示例，
基于 [电商/笔记/支付] 场景，并解释为什么这么写。
```

**示例**：
```
请给我展示一个完整的支付功能 IntentSpec 示例，
基于电商 APP 场景，并解释：
1. 为什么目标部分要包含这些数据？
2. 边缘情况为什么需要考虑这么多场景？
3. 验证方案如何设计才能确保上线成功？
```

**你将获得**：
- 完整案例详解
- 写作原理解释
- 可直接套用的模板

## 常用提示词速查

### PRD 相关

| 需求 | 提示词 |
|------|--------|
| 写 PRD | "请为 [产品] 写一份完整的 PRD，使用用户故事地图组织功能需求" |
| 用户故事 | "请写 5 个核心用户故事，使用 Given-When-Then 格式" |
| 优先级 | "请用 MoSCoW 方法对这些功能进行优先级排序" |
| 非功能需求 | "请补充性能、安全、兼容性方面的非功能需求" |

### IntentSpec 相关

| 需求 | 提示词 |
|------|--------|
| 写 IntentSpec | "请为 [功能] 写一份 IntentSpec，包含完整的边缘情况" |
| 边缘情况 | "请列出这个功能可能遇到的所有异常场景" |
| 验证方案 | "请设计完整的测试方案和上线监控指标" |
| 优化现有 | "当前 [指标] 是 [值]，如何提升到 [目标]？请写 IntentSpec" |

### 方法论相关

| 需求 | 提示词 |
|------|--------|
| RICE 评分 | "请对这 3 个机会进行 RICE 评分" |
| Persona | "请基于这些数据创建 3 个用户画像" |
| JTBD | "请用 JTBD 框架重新描述这个问题" |
| Kano 模型 | "请对这些功能进行 Kano 分类" |

## 文档输出路径

使用本 skill 后，你可以要求将结果保存到指定位置：

```
请把这份 PRD 保存到 [路径]
格式：Markdown
```

**建议的文件结构**：
```
my-product/
├── 01-discovery/
│   ├── opportunity-analysis.md
│   └── user-research.md
├── 02-prd/
│   ├── PRD.md
│   └── user-story-map.md
└── 03-intentspecs/
    ├── feature-a-intentspec.md
    └── feature-b-intentspec.md
```
