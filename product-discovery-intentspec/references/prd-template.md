## 4. 非功能需求（Non-functional Requirements）

### 4.1 性能要求
- 首页加载时间 P95 < （如：1.5秒）
- 核心功能响应时间 P95 < （如：500ms）
- 系统可用性 > （如：99.9%）
- 支持并发用户 （如：10万）

### 4.2 安全要求
- 用户密码加密存储（如：bcrypt）
- 支付数据合规（如：PCI DSS）
- 接口防刷限流

### 4.3 兼容性
- 支持 iOS （版本）+、Android （版本）+
- 支持主流浏览器最新2个版本
- 支持屏幕尺寸：（如：320px - 1920px）

### 4.4 体验要求
- 支持暗黑模式
- 支持无障碍访问（WCAG 2.1 AA）
- 支持多语言（如：中文、英文）

## PRD 编写检查清单

- [ ] 产品愿景是否清晰？（一句话能说明白吗？）
- [ ] 目标用户是否有明确画像？
- [ ] 用户故事是否覆盖完整旅程？
- [ ] 每个用户故事是否有验收标准（Given-When-Then）？
- [ ] 功能优先级是否明确？（P0/P1/P2）
- [ ] 非功能需求是否具体可衡量？
- [ ] 核心功能是否链接到 IntentSpec？
- [ ] 是否有明确的上线成功指标？
