# Product Hunt 产品灵感分析

帮助用户从 Product Hunt 上的成功产品获取灵感，进行深度分析，并生成专业的产品文档。

## 工作流程

### 阶段 1：获取产品信息

用户提供以下任一格式的信息：

**方式 A：产品链接**
```
https://www.producthunt.com/products/[产品名]
```

**方式 B：产品信息摘要**
```
产品名称：XXX
产品描述：XXX
主要功能：XXX
关键评论：XXX
```

**方式 C：多个产品对比**
```
请对比分析以下产品：
1. [产品1]
2. [产品2]
3. [产品3]
```

### 阶段 2：产品分析

分析以下维度：

#### 2.1 产品定位
- 一句话描述
- 声称解决的核心问题
- 目标用户群体（基于描述推测）
- 差异化卖点

#### 2.2 功能分析
- 核心功能列表
- 功能之间的关联
- 用户体验流程（基于描述推测）

#### 2.3 用户反馈分析

**正面反馈（高票评论）**：
- 用户最喜欢什么功能
- 什么价值被反复提及
- 使用场景描述

**负面反馈（机会点）**：
- 提到的缺点和不足
- 用户期望但未满足的需求
- 使用中的痛点

**关键洞察**：
- 从评论中提炼的用户需求
- 产品的潜在改进空间
- 市场机会点

### 阶段 3：差异化机会识别

基于分析，回答：

1. **参考产品做得好的地方**
   - 核心优势是什么
   - 为什么用户喜欢

2. **参考产品的不足**
   - 功能缺失
   - 体验问题
   - 目标用户限制

3. **市场机会**
   - 还有什么需求未被满足
   - 可以服务的不同用户群体
   - 差异化方向建议

### 阶段 4：生成 PRD

生成完整的 PRD 文档，包含：

#### 4.1 产品愿景
- 产品定位（差异化版本）
- 核心价值主张
- 与参考产品的对比

#### 4.2 目标用户
- 用户画像（Persona）
- 用户需求（基于分析推测）
- 典型使用场景

#### 4.3 功能需求
- 用户故事地图
- 功能优先级（P0/P1/P2）
- 用户故事和验收标准

#### 4.4 非功能需求
- 性能要求
- 兼容性要求
- 安全要求

### 阶段 5：生成 IntentSpec（可选）

针对核心功能生成详细规范：

- 目标（基于分析的问题陈述）
- 用户目标（JTBD）
- 成果指标
- 边缘情况
- 验证方案

### 阶段 6：验证建议

提供验证产品方向的行动建议：

- 用户访谈提纲
- MVP 测试方案
- 关键指标监控

## 输出格式

### 产品分析报告结构

```markdown
# 产品分析：[产品名称]

## 1. 产品概述
（基本信息、定位）

## 2. 价值主张分析
（声称解决的问题、目标用户）

## 3. 功能分析
（核心功能、用户体验流程）

## 4. 用户反馈分析
### 4.1 正面反馈
（用户喜欢什么）

### 4.2 负面反馈
（机会点、改进空间）

### 4.3 关键洞察
（需求提炼、市场机会）

## 5. 优缺点总结
| 优点 | 缺点 |
|------|------|
| ... | ... |

## 6. 差异化机会
（你的机会在哪里）
```

### PRD 结构

使用标准的 PRD 模板，包含：
- 产品愿景
- 目标用户（Persona）
- 功能需求（用户故事地图）
- 非功能需求

### IntentSpec 结构

使用标准的五部分格式：
- 目标
- 用户目标
- 成果
- 边缘情况
- 验证

## 分析深度控制

根据用户需求调整分析深度：

**快速分析**（用户只想了解产品）：
- 产品概述
- 优缺点总结
- 关键洞察

**标准分析**（用户想要 PRD）：
- 完整的产品分析
- 完整的 PRD

**深度分析**（用户想要参考开发）：
- 完整的产品分析
- 完整的 PRD
- 核心功能的 IntentSpec
- 验证建议

## 分析原则

### 客观性
- 区分"声称的"和"实际的"
- 区分"事实"和"推测"
- 标注信息来源

### 深度
- 不只看表面功能，分析背后的设计逻辑
- 从评论中提炼深层需求
- 思考"为什么"而不仅是"是什么"

### 实用性
- 提供可执行的建议
- 生成可直接使用的文档
- 给出验证方案

## 使用提示

### 获取最佳结果

1. **提供完整信息**
   - 分享 Product Hunt 链接
   - 或提供产品名称、描述、关键评论

2. **明确你的目标**
   - "我想了解这个产品的设计思路"
   - "我想找到差异化机会"
   - "请生成完整的 PRD"

3. **多问问题**
   - "用户为什么喜欢这个功能？"
   - "这个产品有什么缺点我可以改进？"
   - "我的目标用户和这个产品一样吗？"

### 常见用法

```
"分析这个产品：https://www.producthunt.com/products/xxx"
"对比这三个笔记产品，找出差异化机会"
"基于 Tana 的设计，为我的简化版生成 PRD"
"分析用户评论，找出最大的痛点"
"生成核心功能的 IntentSpec"
```
