# Product Hunt 产品灵感分析助手

从 Product Hunt 上的成功产品获取灵感，分析其设计思路、用户反馈和差异化机会，生成专业的 PRD 和 IntentSpec 文档。

## 使用场景

- 在 Product Hunt 上发现有趣的产品，想深入分析其成功因素
- 寻找产品灵感，了解当前市场趋势
- 学习优秀产品的设计思路
- 基于现有产品寻找差异化机会
- 快速生成产品文档初稿

## 核心能力

1. **产品逆向分析**
   - 分析产品声称解决的问题
   - 解读用户评论（正面和负面）
   - 推测目标用户和使用场景
   - 识别产品优缺点

2. **差异化机会识别**
   - 从负面反馈中发现改进空间
   - 分析未被满足的用户需求
   - 提供差异化定位建议

3. **专业文档生成**
   - 生成完整的 PRD（产品需求文档）
   - 生成核心功能的 IntentSpec（意图工程规范）
   - 提供验证建议

## 输出内容

### 1. 产品分析报告
- 产品定位和价值主张
- 目标用户分析
- 功能拆解
- 用户反馈分析（正面/负面）
- 优缺点总结

### 2. PRD 文档
- 产品愿景
- 目标用户画像
- 用户故事地图
- 功能需求列表
- 非功能需求

### 3. IntentSpec
- 目标和背景
- 用户目标（JTBD）
- 成果指标
- 边缘情况
- 验证方案

### 4. 差异化建议
- 参考产品的不足
- 市场机会点
- 你的差异化定位建议

## 使用建议

### 最佳实践

1. **提供完整信息**
   - 分享 Product Hunt 产品链接
   - 或粘贴产品名称、描述、关键评论

2. **明确你的需求**
   - 是想完全模仿学习？
   - 还是想找到差异化机会？
   - 需要完整的 PRD 还是简单的分析？

3. **深入分析评论**
   - 特别关注 3 星评论（既有优点又有缺点）
   - 注意反复提到的痛点
   - 观察用户的使用场景描述

4. **验证假设**
   - 生成的文档基于公开信息分析
   - 关键假设建议通过用户访谈验证
   - MVP 测试可以验证产品方向

### 示例工作流程

```
第 1 步：在 Product Hunt 浏览，找到感兴趣的产品
第 2 步：复制产品链接或关键信息
第 3 步：使用本 Skill 生成分析报告和 PRD
第 4 步：根据建议制定差异化策略
第 5 步：（可选）进行用户访谈验证假设
第 6 步：（可选）开发 MVP 测试市场
```

## 局限性和注意事项

### Product Hunt 数据的局限性

**1. 幸存者偏差**
你在 Product Hunt 上看到的是成功的产品，大量失败产品没有被展示。高投票数不等于市场成功。

**2. 评论偏向正面**
Product Hunt 社区文化偏鼓励，批评性评论较少。你可能看不到产品的真实问题。

**3. 早期用户偏差**
早期评论者往往是创作者的人脉圈，可能不代表真实目标用户。

**4. 缺乏深度数据**
- 没有留存率数据
- 没有收入数据
- 没有长期使用反馈
- 不知道用户最终放弃的原因

**5. 营销因素干扰**
高投票可能来自营销能力而非产品价值（创始人知名度、发布时机等）。

**6. 市场差异**
Product Hunt 用户（科技从业者、早期采纳者）可能与你的目标用户不同。

### 使用建议

**正确的使用方式**：
```
Product Hunt 产品 → 分析其设计思路 →
了解用户反馈 → 找到差异化机会 →
生成你的 PRD → 验证关键假设 → 开发
```

**错误的使用方式**：
```
Product Hunt 产品 → 直接复制功能 →
开发 → 上线后发现没人用
```

### 降低风险的方法

1. **找多个参考**
   不要只分析一个产品，找 3-5 个同类产品对比

2. **看其他平台的反馈**
   - Reddit 上的真实吐槽
   - App Store/Google Play 的差评
   - Twitter 上的用户反馈

3. **做用户访谈**
   找到 5 个潜在用户，验证：
   - 问题是否真实存在
   - 你的解决方案是否被接受
   - 差异化方向是否正确

4. **MVP 验证**
   先开发最小可用产品测试市场反应，不要直接开发完整功能
