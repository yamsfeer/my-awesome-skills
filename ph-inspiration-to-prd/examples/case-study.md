本示例展示如何分析 Product Hunt 上的 Tana 产品，并生成分析报告和 PRD。

## 产品分析报告

### 1. 产品概述

Tana 是一个融合了笔记、任务管理和数据库功能的工具，核心特色是 "Supertags" 系统，可以自动组织和关联内容。主打 "像你的大脑一样思考"。

### 2. 价值主张分析

**声称解决的问题**：
- 传统笔记工具太僵化，不符合人脑的思维模式
- 信息孤岛，难以建立关联
- 手动整理费时费力

**目标用户**：
- 知识工作者
- 研究员、作家
- 已经用过 Notion/Roam 的高级用户

### 3. 功能分析

**核心功能**：
- Supertags：自动标签和分类系统
- 每日笔记：快速记录想法
- 数据库视图：结构化查看笔记
- 网络图谱：可视化笔记关联

**用户体验流程**：
1. 快速记录想法到每日笔记
2. 通过 Supertags 自动组织
3. 在数据库视图中查看结构化内容
4. 通过图谱发现隐藏关联

### 4. 用户反馈分析

#### 4.1 正面反馈

- **自动关联有价值**："game-changing"、"gets how I think"
- **适合深度工作**："perfect for my research work"
- **概念强大**："powerful"、"Notion meets Roam"

#### 4.2 负面反馈（机会点）

- **学习门槛高**："steep learning curve"、"overwhelming at first"
- **上手困难**："took me a while to understand"
- **价格贵**："too expensive for personal use"
- **缺少移动端**："wish it had mobile app"
- **引导不足**："need better onboarding"
- **协作弱**："collaboration features need work"

#### 4.3 关键洞察

1. **核心价值被认可**：自动关联确实有需求
2. **门槛是最大障碍**：学习成本阻挡了大众市场
3. **价格敏感**：个人用户觉得贵
4. **移动端是刚需**：被多次提及
5. **协作是缺口**：团队使用受限

### 5. 优缺点总结

| 优点 | 缺点 |
|------|------|
| 自动关联功能强大 | 学习曲线陡峭 |
| 适合深度知识管理 | 上手门槛高 |
| 概念创新 | 价格偏贵 |
| 设计美观 | 缺少移动端 |
| 灵活性高 | 协作功能弱 |

### 6. 差异化机会

基于 Tana 的不足，以下方向有机会：

1. **简化版**：保留自动关联，去掉复杂概念，面向普通用户
2. **移动端优先**：填补移动端空白
3. **团队版**：专注协作功能
4. **垂直场景**：针对学生、律师等特定职业
5. **价格友好**：更亲民的定价

## 生成的 PRD（节选）

```markdown
# PRD：SimpleLink（简化版笔记工具）

## 1. 产品愿景

### 1.1 产品定位
一个自动帮你关联笔记的简化版工具，让普通用户也能享受 networked thinking 的价值，无需学习复杂概念。

### 1.2 核心价值主张
- 痛点：笔记越记越多，越难找到关联
- 价值：自动发现笔记间的联系
- 差异化：比 Tana 简单 10 倍，5 分钟上手

## 2. 目标用户

### Persona：大学生小李
- 基本信息：22 岁，大三学生
- 行为：上课记笔记，期末复习时想找关联内容
- 痛点：笔记分散在各处，复习时找不到相关内容
- 目标：轻松找到所有相关笔记

## 3. 功能需求

### 用户故事地图

| 用户活动 | 记录笔记 | 查看关联 | 复习使用 |
|---------|---------|---------|---------|
| 用户任务 | 快速记录 | 查看自动关联 | 按主题复习 |
|         | 导入现有笔记 | 手动添加关联 | 导出复习资料 |
| 功能点 | 快速输入 | 自动关联引擎 | 主题聚合视图 |
|       | 批量导入 | 关联图谱展示 | 导出功能 |

### 核心用户故事

**US-001：自动发现关联笔记**
> 作为大学生，我想看到与当前笔记相关的其他笔记，以便全面复习。

**AC**：
- Given 用户查看一条笔记
- When 向下滑动
- Then 显示"相关笔记"区域
- And 展示最多 5 条相关笔记
- And 显示关联原因

## 4. 非功能需求

- 首页加载时间 P95 < 2s
- 支持 iOS、Android、Web
- 数据加密存储
```

## 生成的 IntentSpec（节选）

```markdown
# IntentSpec：自动发现关联笔记

## 1. 目标

基于 Tana 的反馈，用户需要自动关联功能，但现有方案学习门槛太高。

## 2. 用户目标

作为大学生，我希望：
1. 不用手动整理，也能看到相关笔记
2. 关联是准确的
3. 一眼就能理解为什么相关

## 3. 成果

| 指标 | 目标值 |
|------|--------|
| 关联准确率 | > 70% |
| 用户使用率 | > 50% |

## 4. 边缘情况

| 场景 | 预期行为 |
|------|----------|
| 新笔记无相关内容 | 显示"暂无相关笔记" |
| 关联内容过多 | 只显示最相关的 5 条 |
| 用户觉得关联不准 | 提供反馈按钮 |

## 5. 验证

- 5 个用户访谈确认需求
- MVP 测试关联准确率
```

## 使用建议

基于这个分析，下一步可以：

1. **验证假设**：访谈 5 个大学生，确认他们是否有该需求
2. **MVP 测试**：开发只有自动关联功能的原型
3. **差异化验证**：测试简化版是否比 Tana 更容易上手
