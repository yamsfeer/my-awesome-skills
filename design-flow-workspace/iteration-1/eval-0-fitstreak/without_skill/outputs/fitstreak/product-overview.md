# FitStreak 产品概述

## 一、产品定位

**产品名称：** FitStreak（健身打卡）
**版本：** v1.0
**平台：** iOS + Android（React Native）
**发布目标时间：** 2026 Q3

FitStreak 是一款面向有健身意愿但自律性不足的上班族的运动坚持工具。通过每日打卡、好友挑战、成就激励和进度可视化，帮助用户将运动习惯持续化，解决"三天打鱼两天晒网"的核心痛点。

---

## 二、核心问题

| 痛点 | 现状 | FitStreak 解法 |
|------|------|----------------|
| 缺乏坚持动力 | 没有外部约束，容易放弃 | 连续打卡 Streak + 成就徽章激励 |
| 缺少社交压力 | 独自健身孤独感强 | 好友挑战 + 排行榜 |
| 进度不可见 | 不知道自己进步了多少 | 体重/体脂趋势图 + 打卡日历热力图 |
| 计划难制定 | 不知道怎么安排运动内容 | 智能运动计划模板库 |

---

## 三、目标用户画像

### 主要用户（Primary Persona）：城市上班族健身新手

- **年龄：** 18-35 岁
- **职业：** 白领、互联网从业者、学生
- **健身经验：** 初级到中级（0-2年）
- **可用时间：** 每天 30-60 分钟，多在下班后或早晨
- **设备：** iPhone 或中高端 Android 机
- **痛点：** 自律性不足、缺少监督、不知道自己是否有进步
- **动机：** 减脂塑形、保持健康、社交展示

### 次要用户（Secondary Persona）：运动习惯养成中的健身爱好者

- **年龄：** 25-40 岁
- **特征：** 已有一定运动基础，希望精细化管理进度
- **需求：** 数据追踪、目标挑战、朋友圈竞技

## 四、核心功能清单

### 免费版（Free Tier）

| 功能模块 | 功能点 | 说明 |
|----------|--------|------|
| 每日打卡 | 运动类型选择 | 支持跑步、力量、瑜伽等 20+ 种运动 |
| 每日打卡 | 时长记录 | 手动输入或定时器计时 |
| 每日打卡 | 打卡日历 | 热力图展示连续天数（Streak） |
| 运动计划 | 预设计划模板 | 提供 10+ 套免费计划（3天/周、5天/周） |
| 社交 | 好友系统 | 添加好友、查看好友动态 |
| 社交 | 基础排行榜 | 周/月打卡次数排行 |
| 成就系统 | 基础成就徽章 | 连续打卡 7/30/100 天等里程碑徽章 |
| 数据统计 | 基础统计 | 本周/本月运动次数、时长汇总 |

### 高级版（Premium，¥18/月）

| 功能模块 | 功能点 | 说明 |
|----------|--------|------|
| 体征追踪 | 体重/体脂趋势图 | 折线图，支持历史数据导出 |
| 运动计划 | AI 个性化计划 | 根据目标、时间、水平智能生成 |
| 社交 | 好友挑战赛 | 发起 7/14/30 天挑战，对战形式 |
| 成就系统 | 稀有徽章 | 解锁专属高级徽章与主题皮肤 |
| 数据统计 | 高级报告 | 周/月详细报告，含运动分布、消耗估算 |
| 无广告 | — | 去除所有广告展示位 |

- 付费转化率：8%

## 六、竞品分析

| 产品 | 优势 | 劣势 | FitStreak 差异化 |
|------|------|------|-----------------|
| Keep | 内容丰富、知名度高 | 社交功能弱、偏内容平台 | 更强的社交打卡与 Streak 机制 |
| Nike Training Club | 专业内容、品牌背书 | 缺乏社交竞技 | 轻量化 + 朋友圈挑战 |
| 薄荷健康 | 饮食+运动结合 | 运动追踪不深 | 专注运动坚持，体验更纯粹 |
| Strava | 运动数据强大 | 学习曲线高、偏跑步骑行 | 更适合泛健身、更低门槛 |

## 八、技术架构概览

```
客户端（React Native）
├── iOS App
└── Android App

后端服务
├── API Gateway
├── 用户服务（User Service）
├── 打卡服务（Checkin Service）
├── 社交服务（Social Service）
├── 通知服务（Notification Service）
└── 数据分析服务（Analytics Service）

基础设施
├── 数据库：PostgreSQL（主数据） + Redis（缓存/排行榜）
├── 文件存储：OSS（头像、成就图标）
├── 推送：APNs + FCM
└── 监控：Sentry + DataDog
```
