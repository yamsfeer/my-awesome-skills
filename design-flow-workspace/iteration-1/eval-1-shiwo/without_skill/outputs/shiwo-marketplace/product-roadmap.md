# 拾物 —— 产品路线图

## 版本规划总览

```
Phase 0 (MVP)     Phase 1 (成长)    Phase 2 (扩张)    Phase 3 (商业化)
2026-Q2           2026-Q3/Q4        2027-Q1/Q2        2027-Q3+
|-----------------|-----------------|-----------------|--------------->
核心交易闭环        社区与信任提升      多校联动扩张         收费与增值服务
```

---

## Phase 0 — MVP 核心闭环（2026-Q2，约 8 周）

**目标**：跑通"发布 → 浏览 → 沟通 → 面交"最小可行路径，在 1-2 所高校内冷启动

### 功能清单

| 模块 | 功能 | 优先级 |
|------|------|--------|
| 用户体系 | 微信一键登录 | P0 |
| 用户体系 | 学号/校园邮箱实名认证 | P0 |
| 用户体系 | 个人主页（头像、昵称、学校） | P0 |
| 商品 | 发布商品（拍照+文字+定价+分类） | P0 |
| 商品 | 商品详情页 | P0 |
| 商品 | 下架/删除商品 | P0 |
| 发现 | 首页商品流（按时间排序） | P0 |
| 发现 | 分类导航（教材/数码/生活/服饰/其他） | P0 |
| 发现 | 关键词搜索 | P0 |
| 发现 | 校区筛选 | P0 |
| 沟通 | 微信 IM 私信（基础文字） | P0 |
| 沟通 | 预约面交（时间+地点文字确认） | P0 |
| 交易 | 标记"已售出" | P0 |
| 评价 | 完成交易后基础评价（好评/差评+文字） | P1 |

### 里程碑

- Week 2：原型确认，UI 设计稿交付
- Week 4：前端页面开发完成
- Week 6：后端接口联调完成
- Week 8：灰度上线（目标高校 100 名种子用户）

---

## Phase 1 — 社区与信任提升（2026-Q3/Q4，约 12 周）

**目标**：提升用户信任度与平台活跃度，MAU 达到 1000+

### 功能清单

| 模块 | 功能 | 背景 |
|------|------|------|
| 信任体系 | 诚信评分可视化（星级 + 成交次数） | 增加卖家可信度 |
| 信任体系 | 举报与申诉机制 | 平台合规基础 |
| 信任体系 | 风险交易提醒（超低价、新注册用户） | 降低欺诈风险 |
| 商品 | 图片最多 9 张 | 提升商品展示质量 |
| 商品 | 商品编辑（修改价格/描述） | 卖家体验优化 |
| 商品 | 收藏/想要商品 | 提升买家活跃度 |
| 沟通 | 私信支持图片发送 | 沟通体验提升 |
| 沟通 | 面交地点地图选点 | 降低约定面交成本 |
| 发现 | 价格区间筛选 | 搜索体验优化 |
| 发现 | 按"最近发布"/"最低价"排序 | 搜索体验优化 |
| 通知 | 微信订阅消息（有人私信/商品被收藏） | 提升响应率 |
| 个人中心 | 我发布的 / 我买到的 / 我卖出的 | 交易记录管理 |
| 运营 | 校园推广活动页（开学季/毕业季） | 冷启动活动支撑 |

### 里程碑

- Q3 末：诚信评分系统上线，完成交易 500 笔
- Q4 末：MAU 1000+，覆盖 3 所高校，NPS ≥ 40

---

## Phase 2 — 多校联动扩张（2027-Q1/Q2，约 16 周）

**目标**：批量复制到更多高校，MAU 达到 10000+

### 功能清单

| 模块 | 功能 | 背景 |
|------|------|------|
| 多校支持 | 城市高校圈（同城不同校可交易） | 扩大供需池 |
| 多校支持 | 校园大使招募与管理后台 | 低成本地推 |
| 商品 | AI 辅助定价建议 | 降低定价门槛 |
| 商品 | 商品自动下架（30 天无人问津） | 内容质量维护 |
| 发现 | 热门商品榜单 | 提升发现效率 |
| 发现 | "附近"概念（宿舍楼/教学楼粒度） | 进一步精准匹配 |
| 社区 | 动态广场（晒单/求购帖） | 提升社区氛围 |
| 数据 | 卖家数据看板（浏览量/私信数） | 卖家运营意识培养 |
| 安全 | 身份核验升级（学信网 API 对接） | 提升实名可信度 |

### 里程碑

- Q1 末：覆盖 20 所高校
- Q2 末：MAU 10000+，月成交笔数 5000+

---

## Phase 3 — 商业化与增值服务（2027-Q3+）

**目标**：实现盈利，月 GMV 达到 50 万元+

### 商业化功能

| 功能 | 收费模式 | 预计时间 |
|------|----------|----------|
| 成交服务费 2% | 按成交额抽佣 | 2027-Q3 |
| 商品置顶推广 | CPT（按天收费） | 2027-Q3 |
| 校园广告位 | 开屏/Banner 广告 | 2027-Q4 |
| 会员服务 | 月/年订阅（免服务费+更多图片+优先展示） | 2028-Q1 |
| 品牌店铺 | 校内二手书店/数码维修入驻 | 2028-Q1 |

## 技术债务与非功能性规划

| 阶段 | 重点 |
|------|------|
| MVP | 快速上线，技术选型保守稳健 |
| Phase 1 | 引入自动化测试，规范 CI/CD |
| Phase 2 | 服务拆分（商品服务/IM 服务/用户服务独立部署） |
| Phase 3 | 引入风控系统，对接微信支付担保交易 |

---

## 风险与假设

| 风险 | 等级 | 缓解策略 |
|------|------|----------|
| 冷启动难（双边市场） | 高 | 先做卖家供给侧，邀请宿舍长/学生会种子用户 |
| 学校政策限制 | 中 | 避免宣传"收费"，以"公益互助"定位切入 |
| 用户欺诈/纠纷 | 中 | 实名制 + 评分系统 + 举报机制 |
| 微信审核政策变化 | 低 | 严格遵守微信平台规范，保留备用域名方案 |
