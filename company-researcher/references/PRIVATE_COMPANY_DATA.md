## 非上市公司研究方法

### 方法1：产品体验

1. **下载应用**或使用服务
2. **绘制核心功能**和用户旅程
3. **识别价值主张** - 它解决了什么问题？
4. **与竞争对手比较** - 有什么不同？
5. **评估用户体验** - 是否易于使用？

### 方法2：融资分析

1. **查找所有融资轮次**（天眼查、Crunchbase、36Kr）
2. **计算总融资额**和最新估值
3. **识别投资者** - 是否有知名机构？（红杉、高瓴、腾讯）
4. **计算股权比例** - 投资者获得了多少？
5. **评估烧钱率** - 他们能运营多久？

### 方法3：团队尽职调查

1. **查看LinkedIn**了解创始人背景
2. **搜索之前的公司** - 他们以前成功过吗？
3. **寻找危险信号** - 频繁跳槽、无法解释的空档期
4. **检查大学背景** - 是否有强大的技术背景？
5. **搜索新闻**寻找任何争议

### 方法4：竞争格局

1. **识别直接竞争对手** - 谁解决同样的问题？
2. **识别间接竞争对手** - 替代解决方案
3. **评估市场地位** - 领导者、跟随者还是利基市场？
4. **分析护城河** - 技术、品牌、网络效应、转换成本
5. **绘制生态系统** - 合作伙伴、客户、供应商

### 方法5：用户/市场研究

1. **查看应用商店评论** - 用户喜欢/不喜欢什么？
2. **搜索Reddit/社交媒体** - 真实用户意见
3. **寻找第三方数据** - QuestMobile、SimilarWeb
4. **阅读行业报告** - 市场规模、增长率、趋势
5. **检查专利** - 知识产权保护

## 需要注意的危险信号

### 团队危险信号
- 没有技术联合创始人
- 关键岗位人员流动率高
- 创始人冲突或离职
- 顾问委员会薄弱

### 业务危险信号
- 用户增长下滑
- 高流失率
- 依赖单一客户
- 不可持续的单位经济模型

### 财务危险信号
- 财务数据模糊
- 异常的关联交易
- 估值快速上涨但缺乏明确理由
- 主要投资者退出

### 市场危险信号
- 可寻址市场萎缩
- 新的监管威胁
- 强大的现有竞争对手
- 技术颠覆风险

## 非上市公司估值方法

| 方法 | 何时使用 | 公式 |
|--------|-------------|---------|
| **收入倍数** | SaaS公司 | 收入 × 倍数（非上市公司通常为5-20倍） |
| **现金流折现** | 现金流为正 | 折现未来现金流 |
| **可比交易** | 并购背景 | 查看行业内近期收购 |
| **Berkus方法** | 早期阶段 | 每个里程碑500万-1500万美元 |
| **风险因素叠加** | 早期阶段 | 基础价值 ± 风险调整 |

### 快速估值检查

对于中国AI初创公司（截至2024年）：
| 阶段 | 典型估值 |
|-------|------------------|
| 种子轮 | 1000万-3000万美元 |
| A轮 | 5000万-1.5亿美元 |
| B轮 | 1.5亿-5亿美元 |
| C轮及以后 | 5亿美元以上 |

对于生成式AI公司（溢价）：
| 阶段 | 典型估值 |
|-------|------------------|
| A轮 | 2亿-5亿美元 |
| B轮 | 5亿-10亿美元以上 |

## 非上市公司尽职调查清单

### 投资前

- [ ] 产品演示和个人体验
- [ ] 创始人面试和背景调查
- [ ] 客户参考（与3-5个客户交谈）
- [ ] 竞争分析和市场规模
- [ ] 财务模型审查（如有）
- [ ] 股权表和投资者条款审查
- [ ] 法律和合规审查
- [ ] 知识产权和技术评估
- [ ] 关键人员风险评估

### 向管理层提问

1. 你们解决的核心问题是什么？
2. 如何获取客户，成本是多少？
3. 你们的单位经济模型是什么？
4. 主要竞争对手是谁，差异化优势是什么？
5. 盈利路径是什么？
6. 关键风险是什么，如何缓解？
7. 这笔资金将用于什么？
8. 未来18个月的里程碑是什么？
