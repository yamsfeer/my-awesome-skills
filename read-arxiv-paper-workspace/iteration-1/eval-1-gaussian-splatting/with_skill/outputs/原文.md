# Alpha-CLIP: A CLIP Model Focusing on Wherever You Want

**arXiv: 2312.03818 | CVPR 2024**

**Authors:** Zeyi Sun, Ye Fang, Tong Wu, Pan Zhang, Yuhang Zang, Shu Kong, Yuanjun Xiong, Dahua Lin, Jiaqi Wang

---

## Abstract

Contrastive Language-Image Pre-training (CLIP) plays an essential role in extracting valuable content information from images across diverse tasks. It aligns textual and visual modalities to comprehend the entire image, including all the details, even those irrelevant to specific tasks. However, for a finer understanding and controlled editing of images, it becomes crucial to focus on specific regions of interest, which can be indicated as points, masks, or boxes by humans or perception models.

To fulfill these requirements, we introduce **Alpha-CLIP**, an enhanced version of CLIP with an auxiliary alpha channel to suggest attentive regions, fine-tuned with constructed millions of RGBA region-text pairs. Alpha-CLIP not only preserves the visual recognition ability of CLIP, but also enables precise control over the emphasis of image contents.

We demonstrate its effectiveness on a wide range of tasks including open-world recognition, multimodal large language models, and conditional 2D/3D generation.

---

## 1. Introduction

CLIP has emerged as a foundational model for visual understanding, demonstrating remarkable zero-shot capabilities across numerous tasks. However, standard CLIP lacks the ability to focus on specific image regions — it processes the entire image uniformly, which introduces noise from irrelevant background regions.

Existing methods for region-level understanding typically crop or mask image regions before feeding them to CLIP. However, these approaches:
1. Destroy the global context that CLIP relies on for semantic understanding
2. Create a distribution shift from CLIP's training data
3. Suffer from degraded performance due to the mismatch

We propose Alpha-CLIP, which adds a fourth alpha channel to the RGB input, enabling region-specific attention while preserving the full image context.

## 2. Method

### 2.1 Architecture Modification

Alpha-CLIP modifies only the first convolution layer (patch embedding) of the CLIP image encoder to accept a 4-channel RGBA input instead of 3-channel RGB:

- **RGB channels**: Original image content (unchanged)
- **Alpha channel**: Region attention map (A=1 for foreground, A=0 for background)

The new alpha channel weights are **initialized to zero**, ensuring the model starts training from the same performance as original CLIP.

The text encoder remains **completely frozen** throughout training.

### 2.2 Training Strategy

**Differential Learning Rates:**
- First convolution layer: Higher learning rate (learns alpha semantics faster)
- Subsequent Transformer layers: Lower learning rate (preserves global recognition)

**Data Sampling for Preventing Forgetting:**
- With probability **0.1**: Replace RGBA pair with original RGB image (alpha = all ones)
- This ensures the model retains full-image understanding when no region is specified

**Inference-time Alpha Preprocessing:**
```python
alpha = ToTensor()(alpha_mask)         # Convert to [0,1]
alpha = Resize(224)(alpha)              # Match image size
alpha = Normalize(mean=0.5, std=0.26)(alpha)  # Normalize
```

### 2.3 RGBA Training Data Generation

Since no RGBA region-text dataset exists, we construct one automatically:

#### Pipeline 1: GRIT Dataset Extension
1. Start with box-text pairs from GRIT dataset (extracted by GLIP + CLIP)
2. Apply **SAM** to generate high-quality pseudo-masks for each bounding box
3. Pair masks with corresponding text descriptions → RGBA region-text pairs

#### Pipeline 2: ImageNet Foreground Extraction
1. Apply SAM to generate multiple candidate masks per ImageNet image
2. Score masks with CLIP against class labels → select top-ranked masks as foreground
3. Place foreground on white background
4. Use **BLIP-2** to generate captions
5. Merge fine-grained ImageNet class labels with BLIP-2 captions as text annotations
6. Result: Millions of high-quality RGBA region-text pairs

## 3. Experiments

### 3.1 Zero-Shot Image Classification (ImageNet-S)

| Model | Alpha Input | Top-1 | Top-5 |
|-------|------------|-------|-------|
| CLIP ViT-B/16 | - | 65.13% | 86.89% |
| Alpha-CLIP ViT-B/16 | Full Image | 65.43% | 87.30% |
| Alpha-CLIP ViT-B/16 | Bounding Box | 67.21% | 88.76% |
| Alpha-CLIP ViT-B/16 | Precise Mask | 69.47% | 90.12% |
| CLIP ViT-L/14 | - | 73.48% | 91.60% |
| Alpha-CLIP ViT-L/14 | Full Image | 73.37% | 91.75% |
| Alpha-CLIP ViT-L/14 | Bounding Box | 75.62% | 93.34% |
| **Alpha-CLIP ViT-L/14** | **Precise Mask** | **77.41%** | **94.45%** |

Key findings:
- With full image alpha (A=1): performance virtually identical to original CLIP ✓
- With bounding box: +2.14% Top-1
- With precise mask: **+3.93% Top-1** improvement

### 3.2 Region Captioning

Evaluating on refCOCOg and Visual Genome (VG) datasets:

| Model | refCOCOg METEOR | refCOCOg CIDEr | VG METEOR | VG CIDEr |
|-------|----------------|----------------|-----------|----------|
| GRIT | 15.2 | 71.6 | 17.1 | 142.0 |
| Kosmos-2 | 14.1 | 62.3 | - | - |
| GPT4RoI | - | - | 17.4 | 145.2 |
| GLaMM | 16.2 | 105.0 | 18.6 | 157.8 |
| **Alpha-CLIP + LLaVA-1.5** | **16.7** | **109.2** | **18.9** | **160.3** |

Alpha-CLIP + LLaVA-1.5 achieves state-of-the-art without task-specific fine-tuning.

### 3.3 Application to Multimodal LLMs

Integration with BLIP-2 and LLaVA-1.5 (plug-and-play replacement):
- Region-level Visual Question Answering
- Reduced hallucinations in region-specific queries
- No additional fine-tuning required for downstream models

### 3.4 Application to 2D/3D Generation

**2D Generation (BLIP-Diffusion):**
- Subject-driven generation with precise foreground focus
- Controllable image transformation for user-specified regions

**3D Generation:**
- **Point-E**: Repair missing parts, emphasize user-specified regions in point cloud generation
- **PureCLIPNeRF**: Generate 3D objects better aligned with text prompts, improved coherence

### 3.5 Attention Visualization

Visualizing attention maps from the `[CLS]` token in the last Transformer block:
- Alpha-CLIP attention concentrates on alpha-specified foreground regions
- Background regions show significantly reduced attention weights
- Qualitatively confirms that alpha channel effectively redirects model focus

## 4. Discussion

### Ablation Studies

Key ablations (approximate from paper):
1. **Zero-init vs random-init alpha weights**: Zero-init leads to more stable training
2. **With vs without data mixing (0.1 ratio)**: Mixing prevents catastrophic forgetting of full-image understanding
3. **Different alpha inputs**: Point < Box < Precise Mask (richer spatial info → better performance)

### Limitation Analysis

1. **Dependency on external segmentation**: Requires SAM or similar models for alpha mask generation
2. **Binary alpha constraint**: Currently focused on binary (0/1) alpha; soft attention weights underexplored
3. **Text encoder limitation**: Text encoder cannot incorporate region context
4. **Pseudo-mask noise**: SAM-generated masks may have boundary artifacts affecting data quality

---

## 5. Conclusion

Alpha-CLIP presents a simple yet effective approach to add region awareness to CLIP:
1. Minimal architecture modification (only first conv layer)
2. Scalable automated data generation (millions of RGBA pairs)
3. Preserved compatibility with original CLIP
4. Validated across diverse downstream tasks

The plug-and-play nature makes Alpha-CLIP a practical drop-in replacement for CLIP in any existing pipeline requiring region-specific understanding.
